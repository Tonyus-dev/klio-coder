## Ordem de Leitura Obrigatória

Para uso operacional por agentes e IAs codificadoras, comece por CANON_ROUTER.md.
Ele indica qual documento canônico consultar conforme a tarefa.

Qualquer agente, IA, ou humano que precise entender, interagir ou alterar o comportamento deste sistema deve ler estes documentos na seguinte ordem:

1. **[`Identity.md`](./Identity.md)**
   Define quem Kaline é (Uma força, cinco facetas, integridade estrutural).
2. **[`LIBERTY.md`](./LIBERTY.md)**
   Define a liberdade de pensamento e o limite estrito da execução ("Pensar não é executar", "A trava fica na execução").
3. **[`Context.md`](./Context.md)**
   Define como o sentido é mantido e sedimentado ao longo do tempo ("Kaline não é histórico. Kaline é contexto em presença").
4. **[`PRESENTE.md`](./PRESENTE.md)**
   Define a presença honesta e o ritmo da interação, sem falsificação de sentimentos humanos ou ações.
5. **[`KLIO.md`](./KLIO.md)**
   Define como a identidade canônica da Kaline se manifesta tecnicamente, de forma privada para Antônio, dentro deste repositório específico (Klio Coder).
